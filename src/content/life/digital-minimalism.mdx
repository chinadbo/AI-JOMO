## 数字时代的专注危机

我们每天被数以千计的通知轰炸。Slack、邮件、社交媒体、新闻推送... 注意力成了最稀缺的资源。

三年前，我开始实践**数字极简主义**——不是拒绝技术，而是有意识地选择工具，让技术为我服务，而非控制我。

## 我的核心原则

### 1. 单一用途原则

每个工具只解决一个核心问题：
- Obsidian → 知识管理
- Things 3 → 任务管理
- Raycast → 快速启动
- Arc → 浏览网页

避免 All-in-One 的诱惑，专用工具往往做得更好。

### 2. 深度工作优先

每天上午 9-12 是我的深度工作时间：
- 手机放在另一个房间
- 电脑开启 Focus 模式（屏蔽所有通知）
- 只保留必要的标签页

下午处理协作事务：会议、邮件、Slack。

### 3. 定期断舍离

每月检视一次：
- 这个工具还在使用吗？
- 有没有更简单的替代方案？
- 自动化可以减少哪些手动操作？

## 我的工具箱

### 知识管理：Obsidian

从 Notion 迁移到 Obsidian 是我做过最好的决定之一。本地存储、Markdown、双向链接，这些特性完美契合我的工作流。

我的 Obsidian 结构：

```
📁 01-Inbox      # 临时笔记
📁 02-Projects   # 项目笔记
📁 03-Areas      # 责任领域
📁 04-Resources  # 参考资料
📁 05-Archive    # 归档
```

### 任务管理：Things 3

尝试过 OmniFocus、Todoist、Notion，最终选择 Things 3 的原因是它的**优雅和克制**。

没有复杂的 GTD 系统，简单的 Area → Project → Task 三层结构，配合「今天」「随时」「某天」的时间分类，刚好够用。

### 快速启动：Raycast

替换 Spotlight 的最佳选择。我常用的功能：

- 应用启动
- 剪贴板历史
- 窗口管理
- 自定义脚本

### 写作：iA Writer

写长文时用 iA Writer。专注模式、语法高亮、完美的 Markdown 支持。

### 阅读：Readwise + Reader

Readwise 拯救了我的阅读笔记。所有高亮自动同步，每日回顾功能让知识真正内化。

### 浏览器：Arc

Spaces + Profiles 让我可以清楚地区分工作和个人浏览。Command-T 切换标签页改变了我的浏览习惯。

### 开发：Cursor + Claude Code

AI 辅助编程已经成为我的标配。Cursor 的补全和 Claude Code 的对话式开发，极大提升了编码效率。

## 数字极简的挑战

实践数字极简并非一帆风顺：

**FOMO（错失恐惧症）** — 关闭社交媒体通知初期很焦虑，但一周后明显改善。

**他人期望** — 同事习惯了即时回复，需要重新设定预期（我会在固定时间查看消息）。

**工具迁移成本** — 从一个工具迁移到另一个需要时间适应，不要频繁切换。

## 一个实验

如果你也想尝试数字极简，可以从这个 7 天实验开始：

1. **Day 1-2**：记录所有数字干扰（通知、打断、无目的浏览）
2. **Day 3-4**：关闭所有非必要通知
3. **Day 5-6**：每天只检查社交媒体一次，每次 15 分钟
4. **Day 7**：反思哪些改变对你有效，形成自己的规则
